# Mobile PWA 轻量 IM、审批与预览界面

## 1. 目标

在同一 Next.js 应用中实现 Mobile/PWA 的轻量入口，覆盖 Workspace、Session、消息、审批和预览，不把移动端做成缩小版 Web IDE。

## 2. 绑定需求

- `FR-MOB-001`
- `FR-NOTIFY-001`
- `FR-CHAT-001`
- `FR-RESULT-001`
- `FR-UI-001`

## 3. Mobile UI 模块

| 模块 | 要求 |
| --- | --- |
| Workspace 列表 | 最近工作区、执行域、Connector 状态、待审批数量 |
| Session 列表 | 会话标题、最后消息、Agent 状态、待确认标记 |
| 轻量会话 | 消息流、任务状态、结果摘要、输入框、@ Role Agent |
| 审批详情 | 风险说明、影响范围、批准/拒绝 |
| 预览页 | 预览链接、结果摘要、只读 Diff 或文件摘要 |

## 4. UI 参考

- lobehub：移动会话布局和设置分组。
- codeg：输入框工具条和权限确认动作。

## 5. TDD 与测试计划

### 先写测试

- Mobile 视口 E2E：390x844 下进入 Workspace 列表无横向滚动。
- Session E2E：进入会话后消息、输入框、任务状态不互相遮挡。
- 审批 E2E：审批详情页显示风险、影响范围、批准/拒绝。
- 预览 E2E：预览页展示摘要和打开链接入口。
- 布局断言：底部/顶部导航不遮挡内容，长标题截断或换行。

## 6. 不做

- 不提供本地 Runtime 接入。
- 不选择本地文件夹。
- 不做复杂代码编辑。
- 不做桌面级 Diff 合并。
- 不引入 React Native/Expo 独立端。

## 7. Definition of Done

- [ ] Mobile/PWA 覆盖轻量 IM、审批、预览主路径。
- [ ] 390x844 视口截图稳定。
- [ ] 无横向滚动、无遮挡、无文本溢出。
- [ ] 用户可见文案全部中文。
