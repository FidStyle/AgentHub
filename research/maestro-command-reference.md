## Command Rule

- Prefer `argument-hint` for invocation shape; do not infer parameters from memory.
- For adaptive lifecycle work, start with `/maestro-ralph "<intent>" -y`, then use `/maestro-ralph status` and `/maestro-ralph-execute` / `/maestro-ralph continue` according to session state.
- For a known single stage, use the stage command directly, for example `/maestro-execute`, `/maestro-verify`, or `/quality-review`.
- `.workflow/.maestro/*/status.json` is execution state, not product completion evidence.

## Workflow Paths

- `A_full_project`: brainstorm → blueprint(可选) → analyze 宏观 → roadmap → analyze 微观 → plan → execute → verify
- `B_large_feature`: analyze 宏观 → roadmap → analyze 微观 → plan → execute → verify
- `C_medium_feature`: analyze 宏观 → plan --from analyze:ANL-xxx → execute → verify
- `D_small_fix`: plan → execute → verify
- `E_spec_only`: blueprint → (供人阅读)
- `F_explore_only`: brainstorm → (供人决策)

## Main Pipeline

- Phases: `analyze` -> `plan` -> `execute` -> `verify`
- Quality: `quality-auto-test` -> `quality-review` -> `quality-test`
- Close: `maestro-milestone-audit` -> `maestro-milestone-complete`

## Essential Commands

- `/maestro` <intent> [-y] [-c] [--dry-run] [--super] — 智能协调器 — 根据意图和项目状态自动选择最优命令链
- `/maestro-init` [-y] [--from <source>] — 项目初始化 — 创建 .workflow/ 目录结构和状态文件
- `/maestro-brainstorm` [topic|role-name] [--yes] [--count N] [--session ID] [--update] [--skip-questions] [--include-questions] [--style-skill PKG] — 头脑风暴 — 发散探索，多角色创意，上游起源命令
- `/maestro-blueprint` <idea or @file> [-y] [-c] [--from <source>] — 正式规格文档化 — 7-phase 收敛规格链，上游起源命令，产出 .workflow/blueprint/
- `/maestro-analyze` [phase|topic] [-y] [-c] [-q] [--gaps [ISS-ID]] — 双层分析 — 宏观(文本参数)探索影响面+scope_verdict / 微观(数字参数)Phase 级深入
- `/maestro-roadmap` <requirement> [-y] [-c] [-m progressive|direct|auto] [--from <source>] [--revise [instructions]] [--review] — 路线图编排 — 消费上游 context，纯 Milestone > Phase 分解（不做技术探索）
- `/maestro-plan` [phase] [--collab] [--spec SPEC-xxx] [-y] [--gaps] [--tdd] [--dir <path>] [--from <source>] [--revise [instructions]] [--check <plan-dir>] — 任务规划 — 支持 --from analyze:ANL-xxx 直达、碰撞检测、gap 修复计划
- `/maestro-execute` [phase] [--auto-commit] [--method agent|cli|auto] [--executor <tool>] [--dir <path>] [-y] — 任务执行 — wave 并行执行、delegate 编排、自动提交
- `/maestro-verify` [phase] [--skip-tests] [--skip-antipattern] [--dir <path>] — 验证确认 — 成果检查、gap 发现、下一阶段建议
- `/maestro-quick` [description] [--full] [--discuss] — 快速任务 — 跳过 Phase 管线直接完成，支持 --full/--discuss 模式
